1. 系統架構簡介

PPOV Extractor 是一套專為 Mouldex 模具技術與品質保證設計的自動化數據提取及玻璃化規格數據庫平台。系統核心採用本地服務器架構設計，能大幅簡化現場品質查檢流程。

系統組成：
- 後端服務器 (Backend Server)：基於輕量化 Python Flask 提供製程參數 PDF 實體提取引擎及數據庫 API 操作。
- 前端使用者介面 (Frontend UI)：基於精美莫蘭迪呼吸感玻璃化網頁，支援全響應式多設備操作。
- 打包封裝 (Frozen Binary)：使用 PyInstaller 結合 Inno Setup 進行一鍵化桌面程式安裝。

2. 部署架構與路徑規範

為了解決 Windows 環境下嚴格的資料夾寫入權限問題，系統實施了最嚴謹的「資源與資料分離」原則：

2.1 靜態主程式目錄

當您執行安裝包安裝後，主程式預設安裝於系統資料夾中：

  • 預設路徑：C:\Program Files (x86)\PPOV Extractor\ 或 C:\Program Files\PPOV Extractor\
  • 安裝目錄包含核心執行檔 PPOV-Extractor.exe 及 docs/ 文件夾，此區塊為唯讀保護區，可完美避開非管理員權限遭竄改的風險。

2.2 動態持久化資料目錄

所有需要頻繁寫入與更新的設定、資料庫檔案均統一指向 Windows 用戶專屬的 AppData 漫遊目錄：

  • 動態路徑：%APPDATA%\PPOV-Extractor-Data\ （實際路徑通常為 C:\Users\<使用者名稱>\AppData\Roaming\PPOV-Extractor-Data\）

該目錄下包含以下核心檔案：

檔案名稱 存儲內容 Git/備份 政策
ppov_database.json 存放所有解析並合併後的玻璃化製程規格數據。 重要資料！建議定期備份，卸載軟體時預設保留。
users.json 存放管理員的帳號名稱、角色配置以及安全雜湊密碼。 機密檔案！請勿隨意移動，且避免明文洩漏。
特別注意： 卸載軟體或重新安裝新版本時，系統安裝程式不會刪除 %APPDATA%\PPOV-Extractor-Data\ 目錄，以確保您的歷史數據與修改過的安全密碼 100% 不會丟失。

3. 帳戶權限與密碼安全

系統內建嚴格的角色型存取控制 (RBAC)，區分為兩種操作權限層級：

3.1 品質檢查員模式 (Guest Mode / Inspector)

  • 免密碼驗證：現場工作人員開啟網頁即預設登入品檢員身分，不需輸入任何密碼，防止密碼於現場終端洩露。
  • 核心權限：支援載入現有 Master 規格總表、手動輸入實際檢驗值、以及一鍵匯出符合 A4 規格的「單品 Spec 規格檢驗單」。
  • 視覺優化：為保持現場畫面的清爽度，品檢員模式下右上角的狀態徽章預設完全隱藏，不影響查檢操作。

3.2 系統管理員模式 (Admin Mode)

  • 高權限保護：管理員需點擊右上角登入並輸入安全密碼。
  • 解鎖功能：登入後即可操作 PDF 單檔/資料夾自動同步、規格 CRUD 修改、資料庫清空，以及匯出 Master Excel 總表等管理端點。
  • 預設初始密碼：Admin123 （區分大小寫）。
密碼安全保護： 後端對密碼使用不可逆的 SHA-256 進行安全雜湊儲存。詳細密碼修改指引請參閱同目錄下的 Password_Management.html 文件。

4. 運行防護與疑難排解

4.1 為什麼程式啟動時會出現命令提示字元 (Console) 視窗？

在許多大型工業製造企業（如 Mouldex 內網），Windows 的安全機制與 AppLocker 軟體限制策略非常嚴格。如果將 Python Flask 打包為無視窗後台執行檔，常會因缺乏背景證書或非交互行為被 AppLocker 誤判為威脅進程而強制中斷封鎖。

解決方案： 我們在 PyInstaller 打包配置中開啟了 console=True 模式。這將使系統被安全防護軟體判定為標準的「控制台診斷工具」，能 100% 避開 Windows 阻擋，確保主程式能穩定啟動運作。請在執行時讓該命令視窗在背景運行，切勿手動關閉，關閉該視窗將導致 Flask 後端服務器停止。

4.2 主程式重啟與瀏覽器防雙開機制

系統在啟動時，會自動檢測本地可用 Port，並利用 Timer 延時開啟瀏覽器。系統已內建防雙開機制，能有效杜絕 Flask 重載器子進程重複開啟兩個網頁分頁的情況，確保資源的最高效利用。

4.3 數據備份與手動還原

如果您需要將資料庫遷移至另一台檢驗終端：

  1. 在原終端機輸入路徑 %APPDATA%\PPOV-Extractor-Data\ 並按 Enter 打開目錄。
  2. 複製 ppov_database.json 檔案。
  3. 將該檔案複製到新終端機的相同資料夾目錄中，重啟主程式即可無縫完成規格移轉。
