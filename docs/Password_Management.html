1. 系統帳戶配置說明

PPOV Extractor 系統對不同角色給予相應權限，以確保產線品質操作流暢的同時，數據的安全性也獲得高水準的把關：

  • 帳戶名稱：admin（固定不可修改）
  • 權限等級：System Administrator（系統最高管理權限）
  • 初始預設密碼：Admin123（區分大小寫）
  • 管理範圍：可修改所有型號製程規格的上下限、清空資料庫、執行 PDF 自動同步，以及從後端發送總表備份。
品質檢查員角色（inspector，即訪客模式）為免密碼登入，開啟主畫面即可直接進行產線檢驗，該模式下完全隱藏所有後台管理端點。

2. 密碼加密安全原理

本系統秉持最嚴格的資安規範，後端絕不儲存任何明文密碼。所有密碼的儲存與驗證均採用業界標準的 SHA-256 安全雜湊演算法 (Cryptographic Hash Algorithm)。

2.1 安全雜湊演算法特點

  • 不可逆性 (One-way)：將明文密碼轉換為雜湊金鑰後，即使檔案洩露，亦無法透過任何數學方法逆向反推還原出原始明文密碼，完全杜絕洩密風險。
  • 雪崩效應 (Avalanche Effect)：密碼只要修改任何一個字元（如大小寫），產生的雜湊金鑰將完全改寫，防範彩虹表與暴力破解。

2.2 設定檔結構範例

管理員密碼雜湊值被安全儲存於動態資料夾內的 users.json 檔案中：

{
  "users": [
    {
      "username": "admin",
      "role": "admin",
      "display_name": "系統管理員",
      "password_hash": "3b612c75a7b5048a435fb6ec81e52ff92d6d795a8b5a9c17070f6a63c97a53b2"
    }
  ]
}
上方的雜湊值即代表初始密碼 Admin123。當您修改密碼時，此檔案中的 password_hash 欄位將會自動更新為新密碼的加密雜湊。

3. 忘記密碼與重置密碼

如果系統管理員忘記修改後的密碼，本系統提供了一個極度安全且防禦性極高的本地重置機制：

3.1 重置步驟

  1. 確保您有該台檢驗終端機的 Windows 實體登入權限。
  2. 在 Windows 檔案總管的網址列輸入 %APPDATA%\PPOV-Extractor-Data\ 並按 Enter 開啟目錄。
  3. 物理刪除或重新命名目錄下的 users.json 檔案。
  4. 重啟 PPOV Extractor 主程式。

3.2 安全機制原理解析

當系統在啟動時，會自動檢測 %APPDATA%\PPOV-Extractor-Data\users.json 是否存在。如果檢測到該檔案缺失，後端服務器會自動以初始基線安全地重建一份全新的 users.json，將密碼還原為初始的 Admin123。

資安防護提示： 由於此重置機制要求必須具備該終端機的實體作業系統登入權限，外部網路人員絕對無法跨網絡執行此重置操作，因此在產線終端環境中是非常穩固且安全的實體權限保護設計。

4. 自助式密碼修改步驟

系統管理員可以在登入後，直接透過精美的網頁使用者介面完成密碼變更，無須手動編輯任何代碼或 JSON 檔案：

  1. 開啟 PPOV Extractor 系統，點擊右上角的「管理員登入」。
  2. 輸入使用者名稱 admin 及目前密碼，點擊登入。
  3. 登入成功後，右上角將浮現 [系統管理員 / ADMIN] 狀態徽章。
  4. 點擊徽章右側的「藍色鑰匙🔑圖標按鈕」。
  5. 在彈出的「修改系統管理員密碼」對話框中，依序輸入：
    • 目前使用的舊密碼
    • 欲設定的新密碼（基於強度規範，長度必須大於或等於 6 個字元）
    • 再次確認新密碼
  6. 確認填寫完畢後，點擊「儲存新密碼」。
系統自動確效： 系統會自動驗證所有輸入項。若兩次密碼輸入不一致或少於 6 個字元，表單將會觸發優雅的 Shake 抖動錯誤動畫 提示。修改成功後，將淡入綠色成功通知並於 2 秒後自動關閉 Modal。新密碼將會立即生效並寫入儲存檔。
